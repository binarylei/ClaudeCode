# 00. 目录与阅读指引

> Claude Code 还原源码横跨 CLI / Agent / Tool / MCP / Skill 五大子系统，上百个源文件。本文帮助你回答一个问题：**从何读起，按什么顺序最高效？**

## 关于本书

本书是对 **Claude Code** 的深度源码解析系列。Claude Code 是 Anthropic 官方出品的 AI 编程助手 CLI 工具，我们的分析基于从 source map 还原的源码。

::: tip 以Harness为核心
本书的核心关键词是 **Harness**（缰绳/套具）——围绕大语言模型构建的、使 Agent 能够在特定领域中有效运作的完整基础设施体系。我们不是写"API 使用教程"，也不是写"代码逐行注释"。我们关注的是 **为什么这样设计**——每一个架构决策背后的约束条件、设计权衡和演进逻辑。
:::

**目标读者**

- 具备 1-3 年编程经验的开发者（不限于 TypeScript/Node.js）
- 对 AI Agent / LLM 应用开发感兴趣
- 希望深入理解生产级 Agent 框架内部实现

**四大收获**

1. **架构认知** — 理解 Claude Code 的整体设计思想、模块划分与分层逻辑
2. **源码阅读能力** — 通过逐模块拆解，培养阅读大型 TypeScript 项目的能力
3. **设计模式可迁移** — 每篇提炼 2-4 条可复用的工程模式
4. **Harness 工程思维** — 建立"驾驭 Agent"的系统观，而非"调用 API"的单一视角

**阅读约定**

- **语言**：中文撰写，专业术语保留英文（如 Agent Loop、Token、MCP）
- **代码引用**：精确到文件路径，关键路径引用 10-30 行，非关键部分用 `// 省略 ...` 标注
- **图解**：使用 Mermaid 展示调用链路、数据流、类关系
- **设计要点**：每段源码引用后提炼 2-4 条设计意图

## 博客设计思路

### 为什么以"设计思想"为主线

传统源码分析容易陷入两种极端：

| 极端 | 表现 | 问题 |
|------|------|------|
| 逐行注释 | 每行代码解释"做了什么" | 只见树木不见森林，读完后仍然不知道"为什么" |
| 架构鸟瞰 | 只画几张高层框图 | 缺乏细节支撑，无法落地到实际代码 |

::: tip 关键洞察
**以设计思想为主线，代码和图是论证工具，不是论述主体。** 每篇文章回答的核心问题是"为什么这样设计"，而非"代码做了什么"。删掉所有代码后，剩下的文字仍能讲清核心逻辑——这是我们的质量标准。
:::

### 同心圆组织逻辑

全书以 **QueryEngine（查询引擎）** 为圆心，由内向外逐层展开。每一层有且仅有一个清晰职责：

```
                     ┌─────────────────────────┐
                     │   Part 9-10: 平台与基础设施  │  运行环境（安全、沙箱、UI、远程）
                     │  ┌───────────────────┐    │
                     │  │   Part 5-8: 扩展系统  │    │  知识、上下文、多智能体、权限
                     │  │  ┌─────────────┐   │    │
                     │  │  │ Part 3-4:   │   │    │
                     │  │  │ 工具与命令   │   │    │  能力提供 + 用户入口
                     │  │  │  ┌───────┐  │   │    │
                     │  │  │  │Part 1-2│  │   │    │
                     │  │  │  │架构+引擎│  │   │    │  全景认知 + 核心循环
                     │  │  │  └───────┘  │   │    │
                     │  │  └─────────────┘   │    │
                     │  └───────────────────┘    │
                     └─────────────────────────┘
```

- **内层不依赖外层**：引擎可以脱离具体工具实现独立理解
- **外层扩展内层**：工具系统赋予引擎"手和眼"，平台层提供运行容器

::: tip 关键洞察
这个分层不是拍脑袋分的，而是从源码的 `import` 依赖关系中自然浮现出来的——**依赖方向严格指向圆心**。内层模块的 import 语句中不会出现外层模块的路径，这是全书组织逻辑的唯一硬依据。
:::

### 每篇文章的写作结构

每篇文章遵循五段式结构：

1. **背景介绍** — 一句话定义核心概念 → 对比传统方案建立认知锚点 → 正交分解为 2-4 个子问题
2. **核心逻辑** — 图解先行 → 方案演进路径（从最简单方案逐步引出实际方案）→ 参数设计意图
3. **源码解读** — 核心文件清单 → 完整调用链 → 关键路径引用（引用前概述，引用后提炼）
4. **总结** — 5-8 条一句话结论，诚实标注覆盖边界
5. **参考文献** — 论文、Wiki、官方文档等延伸阅读

### 源码引用纪律

我们不贴全文代码。引用规则：

- **引用前**：1-2 句中文概括该段代码的核心功能
- **引用后**：bullet points 提炼 2-4 条设计要点
- **引用范围**：只引用关键路径，每段 10-30 行
- **省略标注**：非关键部分统一用 `// 省略 ...` 标注，行数精确到实际行号

## 推荐阅读路线

不同目标的读者，可以选择不同的阅读路径。

### ⚡ 快速入门（7 篇）

> 适合：想快速建立全局认知，了解 Claude Code 是什么、如何运转。

```
01 → 02 → 05 → 06 → 09 → 25 → 41
```

这条路线走通 **架构全景 → 核心循环 → 工具基类 → 子智能体 → 设计原则** 的主干链路。读完你能回答：Claude Code 的整体架构是什么？Agent Loop 怎么转？工具怎么注册和调用？多 Agent 怎么协作？

### 🤖 AI 工程路线（9 篇）

> 适合：关注 LLM 对话引擎设计、Agent 编排与知识系统的 AI 工程师。

```
01 → 05 → 06 → 07 → 18 → 21 → 25 → 29 → 41
```

这条路线聚焦 **对话循环 → Token 管理 → 技能系统 → 上下文压缩 → 子智能体 → 权限控制**。核心关注：上下文窗口管理、成本控制、Agent 记忆机制、安全边界。

### 🔧 工具与平台开发路线（8 篇）

> 适合：想为 Agent 框架贡献工具或定制平台能力的开发者。

```
01 → 03 → 09 → 10 → 11 → 12 → 13 → 14
```

从工具基类设计切入，深入理解工具注册、延迟发现、以及行动层/感知层/协调层/隔离层的分层设计。读完能上手开发自定义工具。

### 📚 完整阅读（42 篇）

> 适合：希望系统深入地掌握 Harness 工程全貌的读者。

按章节编号 **01 → 42** 顺序通读。全书按照同心圆结构由内向外展开，顺序阅读能自然建立从核心到外围的认知层次。

## 总结

1. **本书以"设计思想"为主线**——代码和图是论证工具，不是论述主体。每篇文章回答的核心问题是"为什么这样设计"。
2. **同心圆结构内向外展开**——从 QueryEngine 内核到平台基础设施，依赖方向严格指向圆心，内层不依赖外层。
3. **11 个 Part、42 篇文章**覆盖 CLI / Agent / Tool / MCP / Skill 五大子系统，当前已完成 Part 1-2（7 篇）。
4. **四条阅读路线**适配不同目标：快速入门（7 篇）、AI 工程（9 篇）、工具开发（8 篇）、完整阅读（42 篇）。
5. **每篇文章遵循五段式结构**：背景介绍 → 核心逻辑 → 源码解读 → 总结 → 参考文献，确保一致的阅读体验。
6. **源码引用精准到文件路径和行号**，关键路径引用 10-30 行，引用前概述、引用后提炼设计要点。
7. **Part 11 负责"跳出代码"**——将 Claude Code 的具体实现升华为可迁移的 Harness 工程方法论。

> 💡 **阅读建议**：如果你是第一次接触本系列，建议从 [01. 什么是 Agent Harness](../part1/01-什么是Agent-Harness) 开始，再回到本文选择适合你的阅读路线。
