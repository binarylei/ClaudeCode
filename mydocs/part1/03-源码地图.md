## 3. 架构分层与目录映射

### 3.1 七层架构

Claude Code 的源码按调用方向严格分层：**上层调用下层，下层不感知上层**。

| 层 | 目录 | 职责 | 一句话 |
|----|------|------|--------|
| **入口** | `entrypoints/`、`dev-entry.ts`、`main.tsx` | 启动链路 | 程序怎么起来 |
| **引擎** | `QueryEngine.ts`、`query.ts`、`query/` | 对话循环 + 输入分发 | 所有交互的**唯一入口** |
| **指令** | `commands.ts`、`commands/` | 用户指令接口 | `/` 开头的操作入口 |
| **工具** | `Tool.ts`、`tools.ts`、`tools/` | 能力提供 | AI 的手和眼 |
| **智能体** | `skills/`、`tasks/`、`memdir/`、`coordinator/` | 高级编排 | 将工具组合成复杂任务 |
| **服务** | `services/` | 外部系统对接 | API、MCP、OAuth、插件 |
| **平台** | `components/`、`ink/`、`hooks/`、`bridge/`、`sandbox/`、`utils/` | 运行环境与 UI | 安全边界 + 终端渲染 + 工具函数 |

### 3.2 同心圆模型

以引擎为中心，每一层向外扩展——越内越核心，越外越通用：

```
                     ┌──────────────────────────┐
                     │        平台层              │  运行环境（UI、沙箱、远程、工具函数）
                     │  ┌──────────────────┐     │
                     │  │     服务层         │     │  外部系统对接（API、MCP、插件）
                     │  │ ┌──────────────┐ │     │
                     │  │ │   智能体层     │ │     │  高级编排（Agent、Skill、Task）
                     │  │ │ ┌──────────┐ │ │     │
                     │  │ │ │  工具层   │ │ │     │  能力提供（AI 的手和眼）
                     │  │ │ │ ┌──────┐ │ │ │     │
                     │  │ │ │ │指令层 │ │ │ │     │  用户指令接口（/ 命令）
                     │  │ │ │ │┌───┐ │ │ │ │     │
                     │  │ │ │ ││引擎│ │ │ │ │     │  核心：对话循环 + 输入分发
                     │  │ │ │ │└───┘ │ │ │ │     │
                     │  │ │ │ └──────┘ │ │ │     │
                     │  │ │ └──────────┘ │ │     │
                     │  │ └──────────────┘ │     │
                     │  └──────────────────┘     │
                     └──────────────────────────┘
```

### 3.3 分层调用关系

```
入口层    dev-entry.ts → entrypoints/cli.tsx → main.tsx
               │
引擎层    QueryEngine.ts ←→ query.ts ←→ services/api
               │          ←→ commands.ts（指令分发）
               │
指令层    commands.ts → commands/*（80+ 个指令）
               │
工具层    Tool.ts → tools.ts → tools/*（40+ 个工具）
               │
智能体    tasks/  skills/  memdir/  coordinator/
               │
服务层    services/api  services/mcp  services/oauth  services/analytics  services/plugins
               │
平台层    components/  ink/  hooks/  bridge/  sandbox/  remote/
               │
基础层    utils/*  types/*  constants/*  bootstrap/state.ts
```

**关键设计约束**：

1. **调用方向严格自顶向下**：上层可以调用下层，下层不感知上层
2. **引擎是唯一入口**：所有用户输入（自然语言、`/` 指令、工具结果回流）都经过 `QueryEngine`
3. **`bootstrap/state.ts` 是全局叶子**：被所有模块依赖，但它不依赖任何模块

## 5. 依赖地图

### 5.1 依赖分类

`package.json` 中 60+ 个运行依赖，按功能域分组：

**AI 核心**

```
@anthropic-ai/sdk              # Anthropic API 客户端
@anthropic-ai/claude-agent-sdk # Agent SDK（子任务委派）
```

整个项目的基石，所有 AI 对话能力都通过 Anthropic API 提供。

**MCP 协议栈**

```
@modelcontextprotocol/sdk      # MCP 协议 SDK（服务端 + 客户端）
@anthropic-ai/mcpb             # Anthropic 的 MCP Bridge
```

Model Context Protocol 是 Claude Code 连接外部工具和数据源的标准协议，由 `services/mcp/` 提供服务端管理。

**终端 UI**

```
ink                    # 终端 React 框架
react / react-reconciler     # React 核心
chalk                  # 终端文字颜色
cli-boxes              # 终端边框绘制
figures                # 终端 Unicode 符号（✓ ✗ ⚡ 等）
wrap-ansi              # ANSI 文本换行
```

这些包构成了终端 UI 的基础设施层，在 `components/`（406 文件）和 `ink/`（100 文件）中大量使用。

**进程与沙箱**

```
@anthropic-ai/sandbox-runtime # 沙箱运行时
execa                         # 子进程管理
tree-kill                     # 进程树终止
proper-lockfile               # 文件锁
```

Claude Code 需要执行任意 shell 命令，沙箱和安全隔离是核心问题。

**可观测性**

```
@opentelemetry/api             # OTEL API
@opentelemetry/sdk-logs        # 日志 SDK
@opentelemetry/sdk-metrics     # 指标 SDK
@opentelemetry/sdk-trace-base  # 链路追踪 SDK
@growthbook/growthbook         # 特性开关
```

完整的 OpenTelemetry 栈 + GrowthBook 特性开关，用于产品分析、灰度发布和 A/B 测试。

**数据校验**

```
zod                     # TypeScript-first Schema 验证
ajv                     # JSON Schema 验证（更快）
```

两个验证库并存：Zod 用于类型推断场景，AJV 用于高性能 JSON Schema 校验（如工具调用参数）。

**网络与通信**

```
ws                              # WebSocket 客户端
undici                          # HTTP 客户端
axios                           # HTTP 客户端（更丰富的 API）
https-proxy-agent               # HTTPS 代理支持
vscode-jsonrpc                  # JSON-RPC 2.0 协议
vscode-languageserver-protocol  # LSP 协议
```

`ws` + `vscode-jsonrpc` 的组合暗示基于 WebSocket 的 RPC 通信（远程模式），`vscode-languageserver-*` 说明直接与 LSP 服务器通信。

**认证**

```
google-auth-library     # Google OAuth
qrcode                  # 二维码生成
```

OAuth 2.0 登录流程，二维码用于无浏览器终端环境。

**文件与文本工具**

```
diff                    # 文本 diff 生成
marked                  # Markdown 解析
highlight.js            # 代码语法高亮
ignore                  # .gitignore 规则匹配
picomatch               # Glob 模式匹配
fuse.js                 # 模糊搜索
strip-ansi              # ANSI 转义序列移除
```

| Shim 包 | 功能 | 说明 |
|---------|------|------|
| `color-diff-napi` | 终端颜色差异计算 | 语法高亮的增量渲染 |
| `modifiers-napi` | 键盘修饰键状态检测 | 全局快捷键支持 |
| `url-handler-napi` | 系统级 URL 协议处理器 | OAuth 回调等 |

> 这三个 shim 是还原项目中的特殊设计——原始 `.node` 二进制模块无法直接还原，因此用 JavaScript/TypeScript 重新实现。
