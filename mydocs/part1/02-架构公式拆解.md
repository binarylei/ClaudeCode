# 02. 架构公式拆解

## 极简公式

Claude Code 的架构可以用一个极简公式概括：

```
Claude Code = one agent loop
            + tools (bash, read, write, edit, glob, grep, browser...)
            + on-demand skill loading
            + context compression
            + subagent spawning
            + task system with dependency graph
            + team coordination with async mailboxes
            + worktree isolation for parallel execution
            + permission governance
```

这个公式揭示了 Claude Code 作为 Harness 的最核心设计哲学：**不试图成为智能体本身，而是为智能体（Claude 模型）提供一个功能完备的操作环境。**

## 公式逐项拆解

### Agent Loop
核心循环：接收消息 → 调用模型 → 执行工具 → 获取结果 → 再次决策。所有其他机制都建立在这个循环之上。

### Tools
40+ 个原子工具，涵盖文件 I/O、Shell 执行、代码搜索、Web 访问、子智能体生成等。

### On-Demand Skill Loading
按需知识加载系统，避免上下文窗口浪费。

### Context Compression
三层压缩策略：子智能体隔离 → 上下文压缩 → 任务持久化。

### Subagent Spawning
通过 AgentTool 生成独立子智能体，拥有独立的 messages[] 和执行环境。

### Task System
带依赖图的任务管理系统，支持任务的创建、调度、状态追踪。

### Team Coordination
多智能体团队协作，基于 JSONL Mailbox 的异步通信机制。

### Worktree Isolation
基于 Git Worktree 的文件系统隔离，确保并行执行的安全性。

### Permission Governance
多级权限模式 + 细粒度规则控制，构成 Agent 的安全边界。
